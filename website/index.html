About the Project

AI-Powered Blood
Analysis at Scale

This capstone project builds a complete end-to-end machine learning system that takes a patient's 24 normalized blood test values and predicts which of 6 health conditions they likely have — all in milliseconds.

The dataset contains 486 real-world patient records with severe class imbalance (only 5 "Healthy" samples vs 294 Diabetes samples). The system handles this with SMOTE oversampling, stratified splits, and a soft-voting ensemble of the top 3 models.

ML Pipeline

How the System Works

From raw CSV to final ensemble prediction — a carefully designed 7-step pipeline that avoids data leakage and handles class imbalance.

1

Data Loading & Cleaning

Load blood_samples_dataset_test.csv with 486 records × 25 columns. Remap truncated disease names (Thalasse → Thalassemia, Heart Di → Heart Disease) using a disease_map dictionary.

2

Encoding & Train/Test Split

Apply LabelEncoder to the target column. Perform an 80/20 stratified split to preserve class proportions in both splits — crucial for the tiny minority classes.

3

Feature Scaling

StandardScaler fitted exclusively on training data and applied to both train and test sets — preventing any data leakage from the test distribution.

4

SMOTE Oversampling

Apply SMOTE (k_neighbors=3) only on the training set to synthesize minority class samples. k=3 instead of default k=5 because "Healthy" has only 5 original samples.

5

Train 7 Models + Cross-Validation

Train all 7 classifiers (Logistic Regression, KNN, Random Forest, Gradient Boosting, XGBoost, LightGBM, SVM) with StratifiedKFold (k=5) cross-validation to get reliable accuracy estimates.

6

Ensemble The Top 3

Select top 3 models by test accuracy and combine them in a soft-voting VotingClassifier — averaging predicted probabilities for smoother, better-calibrated outputs.

7

Save & Deploy

Persist the winner (ensemble or best individual) as model.pkl alongside scaler.pkl, label_encoder.pkl, and feature_names.json. Load into Streamlit for real-time prediction.

Dataset

Blood Biomarker Data

486 patient records with 24 fully normalized (0.0–1.0) features and no missing values.

📊

Dataset Shape

486 rows × 25 columns — 24 blood biomarker features plus 1 target disease column. All features are pre-normalized to the 0.0–1.0 range.

⚖️

Class Imbalance

Heavily imbalanced — Diabetes has 294 samples while Healthy has only 5 samples. Addressed with SMOTE and stratified splitting.

🧪

No Missing Values

Clean dataset with zero missing entries. No imputation required. Disease labels are cleaned by expanding truncated 8-character names to full names.

ML Models

7 Models. Best 3 Ensembled.

Every model is trained, cross-validated, and ranked. The top 3 by test accuracy are combined into a soft-voting ensemble for the final prediction.

📉
Logistic Regression
Linear baseline classifier with max_iter=1000. Fast and interpretable; establishes the minimum benchmark.
